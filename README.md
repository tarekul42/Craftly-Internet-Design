# Craftly Internet

Craftly Internet is a premium, professional showcase platform designed explicitly for **AI Orchestrators, Builders, and Engineers**. It operates as a stylized social network and portfolio system where users can broadcast architectural logic, fork and integrate nodes, and review developer case studies in a specialized "Technical-Editorial" aesthetic.

## 🧠 How It Works: A Developer's Brief

Imagine if GitHub and a high-end technical magazine had a child, but it lived entirely in a terminal. Craftly treats every piece of content not as a "post," but as a **Logic Node**.

### 1. The Console (The Discovery Layer)
The Console is your primary feed. It's split into two panes:
- **The Stream (Left)**: A continuous list of incoming logic nodes. Think of this as your `git log` of the entire network.
- **The Canvas (Right)**: When you click a node, it expands here. If it's a **Project Node**, you'll see a "Logic Map"—a visual flowchart of how that project's architecture actually works.

### 2. Logic Nodes (The Content Units)
There are two types of data you can broadcast:
- **Experience Logs**: Quick, text-based reflections. Like a `git commit -m "lesson learned"`.
- **Architectural Projects**: Deep-dives into actual builds. These include a Problem/Solution breakdown and a logic extraction map.

### 3. Forking & The Workbench (The Private Layer)
In a regular social media app, you "like" a post. In Craftly, you **Fork** a node.
- When you fork a project, it gets saved to your **Workbench**. 
- This represents "pulling" that logic into your own private knowledge base for later review or integration into your own projects.

### 4. Technical Engagement (The Peer Review Model)
Unlike traditional social apps, engagement here is a form of **Peer Review**:
- **Acks (Acknowledgements)**: A digital "Copy that." It signals comprehension of the logic.
- **Audits**: A high-level validation that the architectural map is sound.
- **Signal**: The more a node is audited and forked, the higher its "Signal" becomes, surfacing it to the top of the network for other engineers.

## 🛠️ Tech Stack

- **Framework**: Next.js (App Router) - Using React 19 and Next.js 15+ features.
- **Styling**: Tailwind CSS - Pure utility-first styling for that rigid, brutalist look.
- **Language**: TypeScript - Strictly typed to ensure the "Logic Map" data structures remain consistent.
- **State**: React Context + LocalStorage - We use the browser's storage as a "Local Database" so your forked nodes persist even if you refresh.

## ❓ Why "Brutalist Engineering" vs. "Regular Social Media"?

You might ask: *Why is everything black and white? Why use mono fonts? Why not just make a regular feed with likes and comments?*

### 1. Signal Over Noise
Regular social media is designed for **engagement** (dopamine hits, colorful distractions, infinite scrolling). Craftly Internet is designed for **orchestration**. By stripping away color and using a rigid grid, your brain focuses entirely on the **architecture** and **logic** of the code being shared.

### 2. Professional Accountability
By replacing "Likes" with "Audits" and "Acks," we remove the performative aspect of social media. Engaging with a post means you are professionally acknowledging its technical validity.

### 3. High-Information Density
Mono fonts and terminal-style layouts allow us to pack more technical data into a smaller space without it feeling cluttered. It speaks the language of the tools we use every day (VS Code, Terminal, Documentation).

### 4. Utility-First Interaction
"Liking" something is passive. "Forking" something is an action. By using terminology borrowed from version control (Fork, Broadcast, Initialize, Terminate), we turn "browsing" into a simulated **engineering workflow**.

## 🚀 Getting Started

```bash
# 1. Clone & Install
bun install

# 2. Launch Console
bun run dev
```

Open [http://localhost:3000](http://localhost:3000) to initialize your identity.

## 🧭 Navigation 

- `/` (Console) - The global stream of orchestrated nodes.
- `/workbench` - Your personal repository of forked nodes.
- `/identity` - Orchestrator profile and network access status.
- `/init` - Broadcast a new node to the network.
